Guía práctica – Semana 3 Día 5

Actividad:
Cliente, servidor, HTTP, GET, POST y primer servidor con Node.js.

Repositorio:
Repositorio dispuesto por el docente.

Importante:
No debes trabajar directamente en main.

Cada estudiante debe crear una rama personal con el formato:

nombres_apellidos

Ejemplo:
maria_gomez
juan_perez
laura_torres_garcia

Evidencia:
La evidencia de esta actividad es únicamente el push de tu rama personal a GitHub.

No debes entregar nada en Moodle.
No debes enviar capturas.
No debes subir archivos adicionales.
No debes publicar enlaces por otro medio.

Comando nuevo del día:
npm start

Pasos obligatorios:

1. Clonar el repositorio

git clone URL_DEL_REPOSITORIO

2. Entrar a la carpeta del proyecto

cd NOMBRE_DEL_REPOSITORIO

3. Crear la rama personal

git checkout -b nombres_apellidos

4. Verificar la rama actual

git branch

La rama marcada con * debe ser tu rama personal, no main.

5. Levantar el servidor

npm start

6. Abrir en el navegador

http://localhost:3000

7. Abrir también

http://localhost:3000/propuestas

8. Probar POST en otra terminal

node cliente-post.js

9. Modificar el mensaje de bienvenida en servidor.js

10. Detener el servidor con Ctrl + C

11. Levantar de nuevo el servidor

npm start

12. Probar nuevamente en el navegador

13. Modificar el mensaje enviado en cliente-post.js

14. Probar POST nuevamente

node cliente-post.js

15. Completar los archivos:

semana3/conceptos-servidor-http.txt
semana3/bitacora-comando-dia5.txt
semana3/reflexion-plataformas-participativas.txt

16. Revisar cambios

git status

17. Agregar cambios

git add .

18. Crear commit

git commit -m "Servidor Node HTTP GET POST semana 3 dia 5"

19. Publicar la rama

git push -u origin nombres_apellidos
