Guía práctica – Semana 3 Día 3

Actividad:
Ciclos con while: repetir procesos de revisión comunitaria.

Repositorio:
Repositorio dispuesto por el docente.

Importante:
No debes trabajar directamente en main.

Cada estudiante debe crear una rama personal con el formato:

nombres_apellidos

Ejemplo:
maria_gomez
juan_perez
laura_torres_garcia

Evidencia:
La evidencia de esta actividad es únicamente el push de tu rama personal a GitHub.

No debes entregar nada en Moodle.
No debes enviar capturas.
No debes subir archivos adicionales.
No debes publicar enlaces por otro medio.

Comando nuevo del día:
npm run practicar

Pasos obligatorios:

1. Clonar el repositorio

git clone URL_DEL_REPOSITORIO

2. Entrar a la carpeta del proyecto

cd NOMBRE_DEL_REPOSITORIO

3. Crear la rama personal

git checkout -b nombres_apellidos

4. Verificar la rama actual

git branch

La rama marcada con * debe ser tu rama personal, no main.

5. Ejecutar el comando nuevo del día

npm run practicar

6. Leer el resultado en la terminal

7. Modificar ciclo-while.js

Por ejemplo:
const maximoIntentos = 5

8. Ejecutar de nuevo

npm run practicar

9. Ejecutar conteo de propuestas

node conteo-propuestas.js

10. Modificar totalPropuestas

Por ejemplo:
const totalPropuestas = 8

11. Ejecutar de nuevo

node conteo-propuestas.js

12. Ejecutar revisión de fuentes

node revision-fuentes.js

13. Modificar totalFuentes

Por ejemplo:
const totalFuentes = 6

14. Ejecutar de nuevo

node revision-fuentes.js

15. Ejecutar seguimiento comunitario

node seguimiento-comunitario.js

16. Modificar semanasDeSeguimiento o propuestasAtendidas

17. Ejecutar de nuevo

node seguimiento-comunitario.js

18. Completar los archivos:

semana3/conceptos-while.txt
semana3/bitacora-comando-dia3.txt
semana3/reflexion-seguimiento-comunitario.txt

19. Revisar cambios

git status

20. Agregar cambios

git add .

21. Crear commit

git commit -m "Ciclos while en JavaScript semana 3 dia 3"

22. Publicar la rama

git push -u origin nombres_apellidos
